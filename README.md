# cnweb1-hth-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式存放，不依赖后端服务，可直接通过浏览器或 GitHub Pages 访问。

## 项目简介

cnweb1-hth-guide 是一个通用型 HTML 页面归档仓库。仓库中的每个页面都是独立完整的文件，彼此之间没有强关联。它们可以被视为一组经过整理的静态文档或参考页面，适合用于展示、分享或历史存档。

## 目录结构

```
.
├── index.html          # 仓库首页 / 导航页（如有）
├── pages/              # 存放各独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
└── assets/             # 共用的静态资源（CSS、JS、图片等）
    ├── style.css
    └── images/
```

- `index.html`：可选，作为仓库的入口页面。
- `pages/`：所有独立 HTML 页面均放置在此目录中，文件名应具备可读性。
- `assets/`：存放多个页面可能共用的样式、脚本或图片资源。

## 页面归档说明

- 每个页面均为完整的 HTML 文档，包含必要的 `<head>` 和 `<body>` 结构。
- 页面之间相互独立，无跨页面依赖（公共资源除外）。
- 归档内容可能随时间更新或删除，不保证长期稳定可用。
- 页面可能包含示例性内容、说明文字或演示性数据，不代表任何实际服务或产品。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意 `.html` 文件。
3. 若通过 GitHub Pages 部署，仓库根目录或 `docs/` 目录下的 `index.html` 将作为默认页面。
